    Motivation

    Why sea level rise?

    Our main question is: How much higher will sea level rise by 2100, and how does that change across coastal regions? We originally explored tsunami risk, but we realized that the data we were using was better suited for showing long-term sea level rise. Tsunamis, storm surge, and flooding are still relevant, but sea level rise is the baseline condition that can make those coastal hazards more severe.

    We wanted the visualization to show both geography and time. A map helps viewers compare coastal countries and ocean basins, while the side panel and line chart show how one selected country changes from 2024 to 2100.

    Design rationale

    Encodings and interactions

    Countries are colored by projected sea level rise in centimeters above the 1995–2014 baseline. We used a yellow-to-red color scale because larger values represent more projected rise, while gray shows countries without projection data in our processed file. This keeps missing data visually separate from low sea level rise.

    The main interactions are basin filtering, metric switching, search, hover/click inspection, zooming, and reset view. Clicking or hovering a country updates the side panel with the country’s values and a D3 line chart comparing that country’s trajectory to the global mean. These interactions make it easier to explore regional differences instead of reading the map as one static image.

    Design alternatives

    What we changed

    We went through several possible designs before choosing the final version. A static line chart alone was not enough because it showed the time trend but lost the geographic pattern. A map alone was also not enough because it showed spatial differences but did not clearly show how sea level rise changes over time.

    We also considered a heatmap with a year slider, but the map plus selected-country line chart felt clearer and more focused for this prototype. Earlier versions used tsunami-risk framing and risk scores, but we moved away from that because the risk calculation and population weighting were not clearly sourced. The final version focuses on sea level rise, with population and elevation treated only as supporting context.

    Data & transformations

    What the data represents

    The visualization uses a processed JSON file generated from CMIP6 sea-level-rise outputs under SSP5-8.5. Sea level rise values are shown in centimeters relative to a 1995–2014 baseline. The country map provides geographic geometry, while the climate values are loaded from the local JSON file.

    We transformed the climate data into a smaller country-level format so it could be loaded by a static D3 webpage. Countries were also grouped by ocean basin so users could filter the map and compare regional patterns. Population and elevation are included as context variables, not CMIP6 variables, so they are separate from the sea level projection data.

    INTERPRETATION

    What to look for

    The most useful comparisons goes beyond which country has the highest number, sepcifically how each country differs from the global mean and from nearby countries in the same ocean basin. Countries in the same basin often move in similar directions, while some places rise faster or slower because regional ocean dynamics add to or reduce the global trend.

    The line chart helps show whether most of the projected rise happens steadily or accelerates later in the century, which matters because a country with moderate rise by 2050 can still face much larger change by 2100.
